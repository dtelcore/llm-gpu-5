# Output Directory

Current state: this directory now stores fresh checkpoints, logs, and probe output for the repaired pipeline. Older artifacts have been moved aside under `output\checkpoints\old` and `output\logs\old`.

This directory stores all training artifacts, model checkpoints, and generated outputs.

## Contents

### Model Checkpoints
- **Trained Weights**: Parameter snapshots saved during training
- **Optimizer State**: AdamW first/second moments for training resume
- **Configuration**: Model hyperparameters and metadata

### Training Logs
- **Loss Curves**: Per-step loss metrics (CSV or JSON)
- **Training Statistics**: Validation perplexity, gradient norms, learning rates
- **Hardware Metrics**: VRAM usage, step timing

### Generated Text
- **Inference Outputs**: Text generated by trained model
- **Prediction Logs**: Token-by-token generation traces

## Saving Checkpoints

After each training step, save model state:

```python
checkpoint = {
    'step': step,
    'weights': model.get_weights(),
    'optimizer_state': model.get_optimizer_state(),
    'config': config.__dict__
}
np.savez(f'output/checkpoints/model_step_{step}.npz', **checkpoint)
```

## Loading Checkpoints

Resume training from checkpoint:

```python
checkpoint = np.load('output/checkpoints/model_final.npz', allow_pickle=True)
model.load_weights(checkpoint['weights'])
model.load_optimizer_state(checkpoint['optimizer_state'])
step = checkpoint['step']
```

## Logging Best Practices

- **Step 0**: Model initialization (random baseline loss)
- **Every N steps**: Save metrics and checkpoint (trade-off between disk I/O and recovery options)
- **Final Step**: Archive best checkpoint with detailed metadata

## Storage Considerations

For extended training:
- **Checkpoint Size**: ~1-10MB per model (depending on parameter count)
- **Log File Size**: ~1KB per step (negligible)
- **Generated Text**: 1-100MB per run (depends on batch size and sequence length)

Periodic cleanup recommended: Keep only last N checkpoints to manage disk usage.
